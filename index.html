Hola chicos, aca dejo lo que entendi del uso de styflux en un breve documento, espero ser clara y precisa, igual cualquier duda o sugerencia me informan... Para poder usarlo debemos escribir styflux -c en nuestra consola....

Config.json

Este archivo se ubica en styflux/config , yo podría llamar a este archivo el cerebrito del styflux ya que aquí se declaran las variables entre otras configuraciones para administrar a tu necesidad el Styflux.

1. "compile" aquí indicamos donde se compilara(renderizara) nuestros estilos:

"compile":{	"ext":".styl",	"cmd":"stylus -c",	"cmdr":"-o",	"route":"../css" // ubicación de carpeta donde se renderizará los estilos}

2. "layout" contiene la suma de los estilos que estan en styflux/view-helpers:

"layouts":{

"active":true,
"path":"./layouts",

"compile":{	"layout.styl":{"helpers":{"fonts":"fonts.styl", // ejm: @font-face font_face("lucidaRegular")"content":"content.styl", // principales contenedores como: #wrapper .box .main"reset":"reset.styl","grids":"grids.styl", // medidas(ancho) basadas en un sistema de grillas"buttons":"buttons.styl","forms":"forms.styl", // etiquetas, clases, id que usamos para los formularios en general"table":"table.styl", // clases asignadas a table tr td"plugins":"plugins.styl","resources":["resources.styl"], // todas las clases reutilizables"hacks":"[@ult]hacks.styl" // clases para navegador IE}}}

3. "views" contiene la suma de los estilos que estan en styflux/views, esto lo usamos en caso de tener 2 tipos de vista para nuestro proyecto, por llamarlo de algún modo "Público" y "Privado":

"views":{	"path":"./views","pathCompile":"./views"}

4. "vars" en esta variable vamos a declarar ....., adicional a ello indicamos la position X/Y de nuestros icon-sets:

"vars":{	"styfluxFonts":"[@url]fonts/", // es la ubicación de carpeta contenedora de las fuentes: .eot .svg .woff	"routImg":"[@url]../img/",	"family1":"'lucidaRegular',Arial,sans-serif", // es recomendable no usar mas de 3 tipos de fuentes	"family2":"'fortunecityRegular',Arial,sans-serif",	"family3":"Arial,sans-serif",	"Styflux":"17px 50%" // en base a un imagen dentro del icon-set declaramos las posiciones x/y en una variable}

Buttons

styflux/view-helpers/buttons : Tendremos el control total de nuestros botones a partir de una clase llamada "btn" y sus tamaños mapeados según diseño.

Btn DefaultBtn SmallBtn MediumBtn Large

Content

styflux/view-helpers/content : Aquí colocamos las clases para nuestros contenedores principales:

#wrapperwidth 980pxmargin 0 auto.row // esta clase es para un contenedor de clases flotantes. *zoom 1&:before,& :afterdisplay tablecontent ""line-height 0&:afterclear both

Fonts

styflux/view-helpers/fonts : Indicamos los tipos de fuente que usaremos en fonts.styl y guardamos las fuentes(.ttf, .eot, .svg, .woff) en css/fonts.

@font-facefont_face ("lucidaRegular") // a esto llamaremos family1@font-facefont_face ("fortunecityRegular") // a esto llamaremos family2
